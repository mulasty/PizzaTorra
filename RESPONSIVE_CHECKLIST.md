# Responsive Checklist — TORRA

## Viewport checklist

### 320 px
- [ ] brak poziomego scrolla
- [ ] H1 jest czytelny i nie łamie się przypadkowo
- [ ] CTA w hero mieszczą się bez nachalnego ścisku
- [ ] sticky bottom bar nie zasłania treści
- [ ] player muzyczny nie koliduje z dolną nawigacją
- [ ] szybkie akcje pod hero są czytelne i klikalne

### 360 px
- [ ] hero wygląda proporcjonalnie
- [ ] przyciski mają wygodny touch target
- [ ] sekcje kart nie są zbyt ciężkie typograficznie
- [ ] kontakt i telefon są łatwe do znalezienia

### 390 px
- [ ] pierwszy ekran pokazuje markę TORRA i komplet głównych akcji
- [ ] brak poziomego overflow
- [ ] szybkie akcje układają się w czytelny grid
- [ ] galeria nie rozbija rytmu strony

### 414 px
- [ ] hero zachowuje premium kompozycję
- [ ] przyciski `Menu / Telefon / Trasa` są łatwe do kliknięcia
- [ ] sekcja kontaktu nie ściska przycisków

### 430 px
- [ ] układ między hero a quick actions wygląda lekko
- [ ] sticky elementy nie dominują ekranu
- [ ] H2 i tytuły kart pozostają czytelne

### 768 px
- [ ] układ tablet pionowo nie wygląda jak rozciągnięty telefon
- [ ] hero ma dobrą hierarchię
- [ ] siatki kart i galerie mają sensowne proporcje

## Device checklist

### iPhone SE
- [ ] hero mieści się bez chaosu
- [ ] CTA są dostępne od razu
- [ ] sticky bar nie zasłania przycisków ani treści

### iPhone 12 / 13 / 14
- [ ] pierwszy ekran prezentuje markę i klimat premium
- [ ] teksty sekcji są wygodne do czytania
- [ ] akcje kontaktowe działają wygodnie kciukiem

### Android średniej klasy
- [ ] strona ładuje się płynnie
- [ ] hero video nie powoduje odczuwalnego spowolnienia
- [ ] przewijanie jest płynne
- [ ] player muzyczny nie psuje ergonomii

### Tablet pionowo
- [ ] układ nie ma pustych, przypadkowych przerw
- [ ] galeria i siatki kart utrzymują dobrą strukturę
- [ ] menu i kontakt są łatwe do odczytania

## Functional checklist

- [ ] `tel:788779853` działa poprawnie
- [ ] link do Google Maps otwiera prawidłową lokalizację lub gotowe zapytanie
- [ ] hamburger menu otwiera się i zamyka poprawnie
- [ ] focus states są widoczne
- [ ] `npm run lint` przechodzi
- [ ] `npm run build` przechodzi
